The booth state endpoints return the live state of an interpretation booth — active interpreter, connected participants, mic status, and stream health. Use this data to power monitoring dashboards, room_coordinator UIs, or any integration that needs to know who is currently active in a booth.

## Endpoints

Voxbento exposes two variants of the booth state endpoint. Use the event-scoped endpoint for all new integrations; the legacy endpoint exists for backwards compatibility.

| Variant | Method + Path |
|---|---|
| Event-scoped *(preferred)* | `GET /api/events/{event_slug}/booths/{language_code}/state` |
| Legacy | `GET /api/booth/{booth_id}/state` |

For the legacy endpoint, construct `booth_id` as `{event_slug}-{language_code}` — for example, `pycon2026-en`.

## Authentication

Both endpoints require a Bearer token in the `Authorization` header. Obtain a token from `POST /api/auth/token`. See the [Authentication guide](../authentication.mdx) for the full token flow.

:::warning
Requests without a valid Bearer token receive a `403 Forbidden` response.
:::

## Path Parameters

**`event_slug`** *(string)* — The event's URL slug, e.g. `pycon2026`. Must be lowercase alphanumeric with optional hyphens, maximum 64 characters.

**`language_code`** *(string)* — ISO 639-1 two-letter language code in lowercase, e.g. `en`. Identifies which language channel's booth to retrieve.

## Example Request

```bash
curl https://voxbento.com/api/events/pycon2026/booths/en/state \
  -H 'Authorization: Bearer eyJ...'
```

## Example Response

```json
{
  "booth_id": "pycon2026-en",
  "event_slug": "pycon2026",
  "language_code": "en",
  "active_interpreter_id": "user-abc123",
  "active_interpreter_name": "Alice",
  "mic_active": true,
  "ingest_connected": true,
  "participants": [
    {"id": "user-abc123", "name": "Alice", "role": "interpreter"},
    {"id": "user-def456", "name": "Bob", "role": "coordinator"}
  ]
}
```

## Response Fields

**`booth_id`** *(string)* — The canonical booth identifier, derived as `{event_slug}-{language_code}`.

**`event_slug`** *(string)* — The event slug this booth belongs to.

**`language_code`** *(string)* — The ISO 639-1 language code for this booth's channel.

**`active_interpreter_id`** *(string | null)* — The participant ID of the interpreter currently authorised to publish audio. `null` when no interpreter is active.

**`active_interpreter_name`** *(string | null)* — The display name of the active interpreter, or `null` if no interpreter is active.

**`mic_active`** *(boolean)* — Whether the active interpreter's microphone is currently unmuted and transmitting audio.

**`ingest_connected`** *(boolean)* — Whether the WHIP ingest connection is currently established and healthy.

**`participants`** *(array)* — List of all participants currently connected to the booth WebSocket.

  <details>
<summary>Participant object fields</summary>


      The participant's unique ID within this booth session.
    **`name`** *(string)* — The participant's display name.
    **`role`** *(string)* — The role granted to this participant: `interpreter`, `room_coordinator`, `event_owner`, or `listener`.

</details>


## Error Responses

| Status | Condition |
|---|---|
| `404 Not Found` | No booth for the given `language_code` exists in memory for this event. |
| `403 Forbidden` | Bearer token is missing or invalid. |

:::info
Booth state is held in memory. A booth enters memory when the first participant joins, and is cleared if all participants leave. A `404` response does not mean the booth is misconfigured — it simply means nobody has connected to it yet.
:::
