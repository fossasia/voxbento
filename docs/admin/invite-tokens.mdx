Invite tokens let you bring participants into a booth without requiring them to register an account. You generate a token for a specific role, copy the join link, and share it with the intended person ‚Äî when they open the link, Voxbento validates the token and drops them directly into the booth with the right permissions. This makes invite tokens ideal for one-time event attendees or interpreters who don't have an existing Voxbento account.

## What Are Invite Tokens?

An invite token is a single-use credential tied to a specific booth and role. It contains:

- The **booth** the token grants access to.
- The **role** the bearer receives when they join ‚Äî `listener`, `interpreter`, or `room_coordinator`.
- An optional **expiry time** after which the token can no longer be used.
- A human-readable **label** you assign to help you track who the token was shared with.

When a participant opens the join URL, Voxbento validates the token and authenticates them directly into that booth with the correct role for the session.

## When to Use Invite Tokens

### Guest Interpreters
Quickly onboard interpreters who don't have a registered account. Generate an interpreter token and share the link ‚Äî they're live in seconds.

### One-Time Attendees
For events where listeners don't need persistent accounts, issue listener tokens and distribute them alongside your event registration materials.

### Room Coordinator Access
Give a user temporary `room_coordinator` access to a specific booth to manage handoffs, without granting them a full `event_owner` role over the entire event.

### Testing Booths
Generate a short-lived token to verify a booth is working before the event starts, then revoke it immediately after.

## Generating a Token

### 1. Open the booth detail page
In the admin panel, navigate to your event ‚Üí **Rooms** ‚Üí select a room ‚Üí **Booths** ‚Üí click the booth you want to generate a token for.

### 2. Find the Invite Tokens section
Scroll down to the **Invite Tokens** section on the booth detail page.

### 3. Fill in the token form
Complete the three fields:

- **Role** ‚Äî select `listener`, `interpreter`, or `room_coordinator` from the dropdown.
- **Label** ‚Äî a short note to help you remember who this token is for, e.g. `Alice ‚Äî French interpreter` or `Press listener seat 12`.
- **Expires in (hours)** ‚Äî how many hours until the token becomes invalid. Leave this blank to create a token that never expires.

### 4. Generate the token
Click **Generate Token**. The new token appears immediately in the table below the form.

### 5. Copy and share the link
Click **Copy Link** next to the new token. The join URL is now in your clipboard ‚Äî share it with the intended participant via email, chat, or any secure channel.

## The Join URL Format

Every invite token maps to a URL in the form:

```text
/join/{token}
```

For example:

```text
https://voxbento.com/join/a3f8c2d1e9b04f7a8c3d2e1f0b5a9c6e
```

When a participant opens this URL, Voxbento:

1. Looks up the token and confirms it has not expired or been revoked.
2. Authenticates the participant with the role, event, and language channel the token was issued for.
3. Redirects them to the correct interpreter or listener page for the booth.

No account creation or password is required.

## Token Expiry

<Tabs>
  <TabItem value="never-expiring-tokens" label="Never-Expiring Tokens">

Leave the **Expires in (hours)** field blank when generating the token. The token remains valid indefinitely until you manually revoke it. Use this for trusted participants who may need to rejoin across multiple sessions.

</TabItem>
  <TabItem value="time-limited-tokens" label="Time-Limited Tokens">

Enter a number of hours in the **Expires in (hours)** field ‚Äî for example, `4` for a token that expires four hours after creation. Voxbento stores the exact expiry timestamp and rejects the token after that point, even if it has never been used.

</TabItem>

## Revoking a Token

To invalidate a token before it is used or before it expires:

1. Find the token in the **Invite Tokens** table on the booth detail page.
2. Click **Revoke** next to the token you want to cancel.
3. The token is immediately marked as used and can no longer be accepted at the join URL.

:::warning
Revoking a token is permanent ‚Äî it cannot be reinstated. If you accidentally revoke the wrong token, generate a new one for the affected participant.
:::

:::info
Share invite links only with the intended participant. Anyone who has the link can join the booth with the assigned role until the token expires or is revoked.
:::
