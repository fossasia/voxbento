  • WebRTC (WHIP/WHEP): UDP-based, browser-native, ultra-low latency (< 500ms). The industry standard for real-time interaction.
  • RTMP: Legacy TCP ingest protocol. Reliable but blocked by many corporate firewalls. Poor browser support for playback.
  • HLS / DASH: HTTP-based chunked streaming. Excellent for VOD and massive broadcasts, but inherently high latency.
  • The WHIP/WHEP Standard: These IETF protocols standardize WebRTC signaling over HTTP, replacing legacy ingest methods like RTMP.

    Why WebRTC for Live Interpretation?

    Live interpretation is a uniquely demanding use case. An interpreter must hear the floor audio, translate it, and transmit it to the audience. If the combined latency of this loop exceeds 1.5 seconds, the attendee will notice a severe mismatch between the visual video of the speaker and the translated audio.

    Because WebRTC utilizes UDP, it can drop lost packets rather than waiting for retransmission (Head-of-Line blocking), which is the primary cause of latency spikes in TCP-based protocols like RTMP and HLS. Furthermore, the WHIP (WebRTC HTTP Ingestion Protocol) and WHEP (WebRTC HTTP Egress Protocol) standards allow standard HTTP requests to negotiate the WebRTC connection, simplifying the signaling architecture significantly.
