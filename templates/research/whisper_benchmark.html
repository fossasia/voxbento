    Benchmarking Framework

    To accurately test a Streaming STT Architecture, we measure the system across three distinct phases of the pipeline:

    1. Ingest Latency: The time from the speaker's microphone to the MediaMTX RTSP server via WebRTC.
    2. Chunking Latency: The time taken by Silero VAD (or similar) to identify a speech segment and dispatch it.
    3. Inference Latency: The time taken by the Whisper model to generate text from the audio buffer.

    Understanding Real-Time Factor (RTF)

    RTF is calculated as Processing Time / Audio Duration. For live interpretation, the system RTF must consistently remain below 1.0. However, to account for concurrent speakers and overlapping audio events, a target RTF of < 0.2 is strongly recommended for production deployments.

    Memory Considerations

    When running local models, VRAM is a strict constraint. A single instance of Whisper large-v3 requires approximately 10GB of VRAM in FP16 precision. When hosting multiple interpretation booths, consider using quantization (e.g., INT8) or smaller models (e.g., small or base) if hardware is limited.
