    Real-Time Audio Protocols

    WebRTC (Web Real-Time Communication)
    An open-source project providing web browsers and mobile applications with real-time communication (RTC) via simple APIs. It utilizes UDP to achieve sub-500ms latency.
    WHIP (WebRTC HTTP Ingestion Protocol)
    An IETF standard (RFC 9610) that uses an HTTP POST request to perform WebRTC signaling for media ingest. It replaces RTMP for browser-based broadcasting.
    WHEP (WebRTC HTTP Egress Protocol)
    An IETF standard for negotiating WebRTC connections to consume media streams. It allows standard HTTP load balancers to distribute viewing traffic.
    SFU (Selective Forwarding Unit)
    A media server routing architecture that receives a single media stream from a publisher and forwards it unmodified to multiple downstream subscribers, minimizing publisher bandwidth.

    Speech Artificial Intelligence

    STT (Speech-to-Text) / ASR (Automatic Speech Recognition)
    The process or technology of converting spoken language audio into written text.
    WER (Word Error Rate)
    The standard metric for measuring the accuracy of an ASR system, calculated by dividing the sum of substitutions, insertions, and deletions by the total number of reference words.
    RTF (Real-Time Factor)
    A metric of processing speed. An RTF of 0.5 means that 10 seconds of audio takes 5 seconds to process. For live streams, RTF must remain strictly below 1.0.
    VAD (Voice Activity Detection)
    An algorithm used to detect the presence or absence of human speech in an audio segment. Critical for splitting continuous streams into manageable chunks for AI inference.

    Live Interpretation

    Simultaneous Interpretation (SI)
    The process of translating speech from one language to another in real-time, typically requiring interpreters to work in pairs and rotate every 15-20 minutes to prevent cognitive fatigue.
    Floor Audio
    The original, untranslated audio feed captured directly from the speaker's microphone at an event.
    Relay Interpretation
    A scenario where an interpreter translates from an already translated feed rather than the floor audio (e.g., Japanese -> English -> French). Requires extremely low latency to remain viable.
