## Findings considered and rejected

The following were identified by subagents, vetted, and rejected — they are
settled design decisions or mis-attributed findings. Do not re-audit them.

| Finding | Reason rejected |
|---------|-----------------|
| `/ws/captions/{booth_id}` has no auth | Documented in `agents.md` as intentional — public caption feed by design |
| `process.terminate()` unawaited (CORRECTNESS-06) | `asyncio.subprocess.Process.terminate()` is synchronous (sends signal); `await wait()` follows correctly in a `try/finally`. The warning in tests is an `AsyncMock` artifact, not production code. |
| `stderr_task` orphaned if subprocess spawn fails (CORRECTNESS-05) | `stderr_task` is created **after** `await asyncio.create_subprocess_exec()` returns (line 56); it cannot be orphaned by a spawn failure on line 46. Mis-attributed. |
| `pkill` command injection via `event_slug` | `validate_event_slug()` enforces `^[a-z0-9]+(?:-[a-z0-9]+)*$` before this code is reached. No vulnerability. |
| Full-state broadcast overhead (PERF-03) | `ConnectionManager.broadcast` sends the message dict as passed — it does not always serialize full booth state. Finding was based on a misread of the broadcast flow. |
| CSRF on forms | SameSite=Lax cookies provide CSRF protection for same-origin form submissions. Adding CSRF tokens would require a new dependency and is low-value given the current threat model. |
