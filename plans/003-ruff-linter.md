## Why this matters

There is no linter or formatter in this repo. `pyproject.toml` has no
`[tool.ruff]` section, no pre-commit config exists, and CI runs no lint step.
Without automated style enforcement, lint-catchable bugs (unused imports,
shadowed variables, bare `except:`) slip through unnoticed and diffs become
noisy. Adding `ruff` (the de-facto standard for FastAPI + Python 3.13 projects)
gives instant feedback with zero config overhead.

## Steps

### Step 1: Add ruff to dev dependencies and configure pyproject.toml

Add `"ruff>=0.9.0"` to the `dev` dependency group in `pyproject.toml`:

```toml
[dependency-groups]
dev = [
    "pytest==9.0.3",
    "anyio>=4.0.0",
    "ruff>=0.9.0",
]
```

Add ruff configuration sections at the end of `pyproject.toml`:

```toml
[tool.ruff]
target-version = "py313"
line-length = 120

[tool.ruff.lint]
# E/W: pycodestyle, F: pyflakes, I: isort, C4: flake8-comprehensions
select = ["E", "W", "F", "I", "C4"]
ignore = [
    "E501",   # line too long — handled by formatter, not linter
    "E402",   # module level import not at top — some portal files use deferred imports
]

[tool.ruff.lint.per-file-ignores]
"tests/*" = ["F401"]  # allow unused imports in test files (fixtures, re-exports)
"alembic/*" = ["F401", "E402"]
```

**Verify**: `grep -c '\[tool.ruff\]' pyproject.toml` → 1

### Step 2: Sync dependencies to install ruff

```bash
uv sync --all-groups
```

**Verify**: `uv run ruff --version` → prints a version string like `ruff 0.9.x`

### Step 3: Run ruff in check mode and assess violations

```bash
uv run ruff check . 2>&1 | tee /tmp/ruff-initial.txt
wc -l /tmp/ruff-initial.txt
```

Review the output. If there are:
- **0 violations**: proceed directly to Step 5.
- **Fewer than 200 violations**: proceed to Step 4 (auto-fix).
- **200 or more violations**: treat as a STOP condition — report the count and
  the top 10 violation types (`uv run ruff check . --statistics`) before doing anything.

**Verify**: command exits 0 or 1 (1 means violations found — that is expected).

### Step 4: Apply auto-fixes

```bash
uv run ruff check --fix .
uv run ruff check . 2>&1 | tee /tmp/ruff-after-fix.txt
```

Review `/tmp/ruff-after-fix.txt`. Only auto-fixable issues should remain fixed.
If any changes were made to files, review `git diff` before committing.

**Verify**: `uv run ruff check .` exits 0 (no remaining violations) OR the
remaining violations are all `E`/`W` style issues that require manual decision.
If remaining violations require manual edits beyond style, report them and do
not fix them in this plan.

### Step 5: Add ruff format check (optional — only if codebase is already formatted)

Run `uv run ruff format --check .`. If this reports many files need formatting,
**skip the format step** and note it in the plan README. If it reports 0 or
fewer than 10 files, run `uv run ruff format .` to apply formatting.

Do not run `ruff format` if it would touch more than 10 files — that is a
separate cleanup task.

**Verify**: decision recorded in this plan's status section.

### Step 6: Add lint step to CI

In `.github/workflows/tests.yml`, in the `unit-tests` job, add a lint step
**after** the "Sync dependencies" step and **before** "Validate JS syntax":

```yaml
      - name: Lint (ruff)
        run: uv run ruff check .
```

**Verify**: `grep -A1 'Lint (ruff)' .github/workflows/tests.yml` → shows `run: uv run ruff check .`

### Step 7: Create .pre-commit-config.yaml

Create `.pre-commit-config.yaml` in the repo root:

```yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.9.0   # pin to same major as pyproject.toml dep
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format
```

**Verify**: `cat .pre-commit-config.yaml` shows the above content.

### Step 8: Run full test suite to confirm nothing is broken

```bash
API_KEY_ENCRYPTION_KEY="ci-test-encryption-key-must-be-32-chars-long" uv run pytest tests/ -q
```

**Verify**: exits 0, same pass count (385 or more).

## Maintenance notes

- The `rev:` pin in `.pre-commit-config.yaml` should be bumped when `ruff` is
  upgraded in `pyproject.toml`.
- `E402` (module-level import not at top) is ignored globally because several
  portal modules use deferred imports inside functions. If those are ever
  refactored to top-level imports, remove the `E402` ignore.
- Format-only (`ruff format`) is kept separate from lint (`ruff check`); apply
  format in a dedicated cleanup PR to avoid giant diffs mixed with feature work.
